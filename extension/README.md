# PeekGuard — 숄더서핑 감지 크롬 확장

공공장소에서 **화면을 응시하는 제3자**를 웹캠으로 감지해, 화면을 즉시 가리고
**"왜 가렸는지"를 신호·점수·밴드로 투명하게** 보여주는 온디바이스 보안 도구.

> **정직한 포지셔닝:** 숄더서핑 감지 자체는 새로운 아이디어가 아니다(상용 앱·특허 존재).
> 이 프로젝트의 차별점은 ① **판정 근거의 완전한 투명화**(qshing-detector 와 동일한
> 신호+가중치+밴드 결정엔진), ② **초경량 온디바이스 스택**(감지기 전체 245KB, TF.js 불필요),
> ③ **오탐을 1급 설계 목표로 다루는 히스테리시스**다. 학습·포트폴리오 프로젝트로 만든다.

---

## 동작 원리

```
[웹캠 320×240] ──6fps──▶ [240px 다운스케일·그레이스케일]
                              │
                        [pico.js 얼굴 감지]        ◀ 245KB 순수 JS, wasm/TF.js 없음
                              │  (좌표·크기·품질만)
                        [결정엔진 engine.js]        ◀ 순수 함수, 단위테스트 12종
                              │  신호 → 가중치 점수 → 밴드
             ┌────────────────┼────────────────┐
        safe(배지 녹색)   caution(배지 !)   danger(블러/디코이 + 근거 카드)
```

### 판정 신호 (투명 공개 — 오버레이에 그대로 표시)

| 신호 | 점수 | 심각도 | 의미 |
|------|------|--------|------|
| `extraFace` | 35 | high | 사용자 외 얼굴이 화면 방향을 향해 감지 |
| `persistentGaze` | 30 | high | 그 얼굴이 0.5초+ 지속 (스쳐감 아님) |
| `closeRange` | 20 | medium | 화면을 읽을 수 있는 근거리 |
| `approaching` | 15 | medium | 얼굴 크기 증가 — 접근 중 |
| `crowd` | 15 | medium | 주변 3명 이상 |
| `userAbsent` | 0 | info | 사용자 미감지(참고) |

**밴드:** safe(<30) · caution(30–59, 배지만) · **danger(≥60, 화면 보호)**

핵심 조합: 스쳐 지나가는 행인 = `extraFace` 35점 → **caution, 화면 안 가림**.
지속 응시자 = 35+30 = 65점 → **danger, 즉시 블러**. 이것이 오탐 억제의 본질이다.

### 오탐 억제 (설계 1급 목표)

- **발동 히스테리시스:** 0.5초(3프레임) 이상 지속돼야 `persistentGaze`
- **해제 히스테리시스:** 위협 소실 후 2초간 깨끗해야 해제 → 깜빡임 방지
- **1프레임 감지 실패 허용:** 트랙 유지 (감지기 노이즈에 강건)
- **시선 프록시:** pico 는 정면(≈카메라=화면 방향) 얼굴에 강함 → 옆을 보며
  지나가는 사람은 애초에 잘 안 잡힌다. 불확실한 "정밀 시선각" 대신 이 특성을 신호로 사용.
- 민감도 프리셋: 둔감(혼잡 장소) / 보통 / 민감(금융 작업)

### 프라이버시

- 영상·프레임은 **저장·전송 0** — 캔버스에서 분석 즉시 폐기
- 확장은 **네트워크 요청이 전혀 없음** (오프라인 동작)
- 로그는 위협 이벤트의 **시각·점수·신호 id·지속시간** 메타데이터만, `chrome.storage.local`

## 설치 & 테스트

1. `chrome://extensions` → 우상단 **개발자 모드** ON
2. **압축해제된 확장 프로그램 로드** → 이 폴더 선택
3. 설치 직후 열리는 페이지에서 **카메라 권한 허용** (1회)
4. 툴바 PeekGuard 아이콘 → 토글 ON
5. 테스트: 아무 http/https 페이지를 연 뒤, **다른 사람이 뒤/옆에서 화면을 1초쯤 바라보게**
   한다 → 블러 + 근거 카드. 혼자일 땐 배지가 녹색(안전) 유지.
   - 혼자 테스트하려면: 휴대폰에 얼굴 사진을 띄워 웹캠에 비추면 두 번째 얼굴로 인식된다.

### 개발 테스트 (Node)

```bash
node test_engine.mjs   # 결정엔진 시나리오 12종
node test_pico.mjs     # 실제 인물 사진 감지 / 노이즈 오탐 0 검증
```

## 아키텍처 (MV3)

| 파일 | 역할 |
|------|------|
| `engine.js` | ★ 결정엔진. 순수 함수 — 신호/가중치/밴드/히스테리시스. Node 테스트 가능 |
| `offscreen.html/js` | 웹캠 보유 + pico 감지 루프(6fps). MV3 서비스워커는 getUserMedia 불가하므로 offscreen 문서 사용 |
| `background.js` | 상태 소유·중계만. offscreen 수명주기, 배지, 로그, 탭 브로드캐스트 |
| `content.js` | 오버레이(블러+근거 카드 / 디코이). closed Shadow DOM 으로 사이트 간섭 차단 |
| `popup.*` | 상태·토글·민감도·모드·일시정지·이벤트 로그 (표시 전용) |
| `permission.*` | 카메라 권한 1회 부여 (offscreen 은 권한 프롬프트를 못 띄움) |
| `lib/pico.js` | 얼굴 감지기 (MIT, nenadmarkus/picojs) |
| `models/facefinder` | pico 캐스케이드 (234KB) |

## 한계 (정직하게)

- **전면 카메라 화각 밖은 못 본다.** 정후방 "어깨 바로 뒤"는 노트북 카메라의 사각지대.
  옆·대각선·카메라 시야 내 후방만 커버. 완전한 방어는 물리 필터 병용.
- 저조도·역광에서 감지율 저하. 안경 반사·마스크는 영향 적음(얼굴 윤곽 기반).
- 화이트리스트(동료 얼굴 등록)는 관련 특허(US 11,244,148 등) 영역과 겹쳐 **의도적으로 제외**.
  일시정지(5분)로 대체.
- 유사 상용품 존재(EyesOff 등) — 본 프로젝트는 상용이 아닌 학습·포트폴리오 목적.
